Data Governance

The SPROUT project implements robust data governance practices to ensure ethical use of speech data and to protect participant privacy and rights.

1. Ethical Framework

Our ethical framework is built on principles of respect for persons, beneficence, justice, and transparency.

Ethical Principles and Protections

Core Ethical Principles

  • Respect for Persons: Recognizing the autonomy of individuals and protecting those with diminished autonomy
  • Beneficence: Maximizing benefits while minimizing potential harms
  • Justice: Fair distribution of benefits and burdens of research
  • Transparency: Clear communication about data collection, usage, and sharing

Implemented Protections

  • IRB Approval: All research protocols approved by Institutional Review Boards at participating institutions
  • Informed Consent: Comprehensive consent process with clear explanation of:
    • Purpose of data collection
    • Types of data collected
    • How data will be used
    • Potential risks and benefits
    • Data sharing options
  • Tiered Consent: Participants can choose different levels of data sharing:
    • Research use only
    • Non-commercial development
    • Commercial development with restrictions
  • Right to Withdraw: Participants can withdraw consent and request data deletion

2. Data Access Controls

Access to the SPROUT dataset is managed through a controlled access mechanism to ensure appropriate use while protecting participant privacy.

Access Request Process

Access Request Workflow

  1. Registration: Researchers register on the SPROUT data portal
  2. Data Access Request Form: Submission includes:
    • Research purpose and objectives
    • Specific data elements requested
    • Data security measures
    • Institutional affiliation
    • Funding sources
  3. Review by Data Access Committee: Evaluation based on:
    • Scientific merit
    • Ethical considerations
    • Alignment with consent provisions
    • Data security protocols
  4. Data Use Agreement (DUA): Legal contract specifying:
    • Permitted uses
    • Prohibition on re-identification attempts
    • Data security requirements
    • Publication requirements
    • Term and termination conditions
  5. Institutional Signature: Requires sign-off from authorized institutional official
  6. Access Provisioning: Secure credentials provided for data access

Access Levels

Access Level Description Requirements
Open Access De-identified metadata and summary statistics Registration only
Restricted Access De-identified individual-level data Approved DAR and signed DUA
Controlled Access Audio recordings and sensitive data Approved DAR, signed DUA, and additional security measures

3. Data Security Measures

Comprehensive security controls protect data throughout its lifecycle, from collection to storage and access.

Security Infrastructure

Technical Security Controls

  • Encryption:
    • Data encrypted at rest using AES-256
    • Transport encryption using TLS 1.3
    • Key management with regular rotation
  • Access Controls:
    • Multi-factor authentication
    • Role-based access control
    • Principle of least privilege
    • Regular access reviews
  • Monitoring and Auditing:
    • Comprehensive audit logging
    • Automated anomaly detection
    • Regular security assessments

Security Compliance

  • HIPAA compliance for applicable data
  • NIST Cybersecurity Framework
  • Regular third-party security assessments
  • Incident response plan with notification procedures

4. FAIR Data Principles

The SPROUT dataset adheres to FAIR data principles to maximize the value and reuse potential of the data.

FAIR Implementation

Findable

  • Persistent Identifiers: DOIs assigned to datasets and versions
  • Rich Metadata: Comprehensive metadata following established schemas
  • Indexing: Metadata indexed in relevant research data repositories

Accessible

  • Standardized Protocol: Data retrievable via standard HTTPS
  • Authentication: Clear authentication process for restricted data
  • Metadata Availability: Metadata remains accessible even if data access is restricted

Interoperable

  • Standard Formats: Data provided in widely-used formats (WAV, CSV, JSON)
  • Controlled Vocabularies: Use of standard terminologies and ontologies
  • Linked Data: References to related datasets and resources

Reusable

  • Clear Licensing: Explicit data usage licenses
  • Provenance: Detailed documentation of data collection and processing
  • Community Standards: Adherence to domain-specific data standards

5. Data Citation and Attribution

Proper citation of the SPROUT dataset is required in all publications and derivative works.

Citation Guidelines

Required Citation Format

Bensoussan, Y., Ghosh, S. S., Rameau, A., Boyer, M., Bahr, R., Watts, S., Rudzicz, F., Bolser, D., Lerner-Ellis, J., Awan, S., Powell, M. E., Belisle-Pipon, J.-C., Ravitsky, V., Johnson, A., Zisimopoulos, P., Tang, J., Sigaras, A., Elemento, O., Dorr, D., … Bridge2AI-Voice. (2024). SPROUT Dataset [version]. Zenodo. https://doi.org/[DOI]

Acknowledgment Text

In addition to the citation, please include the following acknowledgment in the publication:

"This research utilized the SPROUT dataset, developed by the Bridge2AI-Voice consortium with funding from the NIH Common Fund's Bridge to Artificial Intelligence (Bridge2AI) Program."

Version-Specific Citation

Always cite the specific version of the dataset used in your research. Each version has a unique DOI.

Derivative Works

If creating derivative datasets or models based on SPROUT data:

  • Clearly indicate the relationship to the original SPROUT dataset
  • Describe any modifications or additions made
  • Include the required citation and acknowledgment
  • Provide access to derivative code when possible
