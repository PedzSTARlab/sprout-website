Data Governance

Accessing the Data

The SPROUT raw audio and video datasets, metadata, and survey data (json and tsv) are available for request. Datasets are governed by Data Use Agreements (DUAs), appropriate for the sensitivity of their respective content. Data access is subject to approval by the governance committee.

Data is stored on Northwestern University's secure infrastructure with three registered access levels and their requirements:

Credentialed Access

Description: Full data repository which includes raw audio and video recordings, transcripts, surveys including additional detailed demographic information (e.g., caregiver education, household income, multilingual exposure) and complete test results. Access is available only under heightened security controls, with IRB approval and explicit institutional review. This dataset is available under a Data Use Agreement for non-commercial research and educational use.

Requirements: Submitted request form, verified institution credentials, research proposal abstract, IRB-approved protocol, a lab-provided Northwestern affiliated NetID, and a Globus End Point.

Secured Access

Description: A curated subset of de-identified audio data and basic demographic variables. This dataset is available under a Data Use Agreement for non-commercial research, technology development and educational use.

Requirements: Request form submission, project abstract, a Globus End Point, adherence to Creative Commons Attribution-NonCommercial 4.0 International license.

Registered Access

Description: A curated subset of de-identified audio data and basic demographic variables for participants who have explicitly consented to commercial use. These data are intended for community reuse, including AI model development in academic and commercial domains.

Requirements: Registration and a Data Use Agreement that restricts commercial development to educational, therapeutic, or entertainment technologies for children.

De-Identification Levels

Since audio recordings are considered voice prints, we begin deidentification at the recording session level. We take a layered approach to data protection to balance transparency with privacy.

  • All Personal Identifiable Information is stored securely in REDCap, with limited access to specified personnel and is not located on the data storage server.
  • All data on the storage server are de-identified with participant codes at the source of the recording.
  • Since young children require some redirection for engagement, the childâ€™s first name only may be used in the recording. All data are de-identified and stored using abstract participant IDs.
  • Only key demographic variables (city, race, age, biosex, risk status) are available in the Secured Access and Registered Access dataset.

Consent

Does this dataset allow only the non-commercial use of the data?
Yes, currently the data can be used for non-commercial use only. We will notify registered users when we have dataset updates.
Does this dataset allow only the use of the data in a specific geographic location?
No
Does this dataset allow only the use of the data for a specific type of research?
No
Does this dataset allow only the use of the data for research that does not involve the development of methods or algorithms?
No. The data can be used to develop algorithms used for educational, therapeutic, child entertainment tools but at this time they cannot be distributed for commercial purposes.

Consent Details

Data that does not contain direct identifiers will be shared with external users for future research through a secure database. Data that poses a low risk of causing individual re-identification will be shared through secured access and use agreements with the public. Data that poses a heightened risk of re-identification if shared in public access datasets will be shared through a controlled access mechanism with authorized researchers.

Data Access

If you are interested in accessing the SPROUT datasets, please submit an application through our secure REDCap system. All requests will be reviewed by the governance committee to ensure compliance with our Data Use Agreements and ethical standards.
